肖恩技术周刊（第 87 期）：青少年社交媒体禁令

对周内阅读的技术内容精品（个人向）进行总结。觉得不错可点击上方订阅，第一时间获取更新通知。

全球关于青少年社交媒体禁令浪潮正在蔓延：

  • 澳大利亚：2025年12月起禁止16岁以下儿童在TikTok、Instagram、Snapchat、YouTube、Facebook/Threads、X、Reddit等平台注册或持有账户
  • 法国：2026年1月通过法案禁止15岁以下使用社交媒体，计划于2026年9月新学年实施，同时禁止高中校园内使用手机
  • 挪威：正在推进15岁年龄限制的立法提案
  • 丹麦：2025年秋季达成协议，计划2026年中期实施15岁以下禁令，允许13-15岁在家长授权下使用
  • 英国：2024年通过《在线安全法》（Online Safety Act），要求平台实施年龄验证和内容过滤
  • 意大利：要求14岁以下儿童创建社交媒体账号需取得家长许可
  • 德国：要求13至16岁青少年使用社交媒体须经家长许可
  • 马来西亚：2025年通过《网络安全法令》，计划2026年实施16岁以下禁令

禁令背后的核心动因

心理健康危机与社交媒体的高度重合

过去十年，全球青少年抑郁、焦虑、自残率显著上升，尤其在女孩、性少数群体和社交媒体重度用户中更明显。一些数据：

  • 澳大利亚政府电子安全机构调查显示96%的10-15岁儿童使用过至少一个社交媒体平台
  • 约70%的青少年报告曾接触有害内容（暴力、仇恨/厌女内容、危险挑战、自残内容等）
  • 美国心理学会研究发现，在高频率社交媒体使用者中，41%认为自己的心理健康状况差或很差

发育科学视角

神经科学研究表明，人类大脑前额叶皮层（负责自我控制）到18-25岁才发育成熟。社交媒体的算法逻辑是不断推送高刺激内容、诱导互动与停留，这可能与青少年尚在发育中的自控能力产生冲突。

也存在质疑声音

  • 美国国家科学院2024年报告指出现有研究多为相关性而非因果关系，心理健康变迁还受教育压力、家庭因素、经济环境等多重变量影响
  • 此举可能侵犯儿童的表达自由、信息获取权、文化参与权等
  • 弱势群体往往在网络上比线下更容易寻求支持，强制断网可能加剧孤立与求助障碍
  • 年龄验证系统极易被绕过，并且多数家长愿意帮助孩子规避，导致禁令难以有效实施
  • 可能把孩子推向更隐蔽、更危险的网络空间
  • Meta等平台建议激励全行业提供安全、隐私保护、适龄的在线体验，而非一刀切禁令

总的来说，这一趋势正将问题从家庭层面问题从家庭层面扩展到国家立法层面，促使全球重新思考：在算法驱动的注意力经济中，如何更好地保护青少年群体？

今日观察回顾

石英、机械与大模型

技术趋同后，品牌和价值解释成为核心竞争力。瑞士钟表业从精度竞争转向奢侈品叙事，成功转型。AI大模型领域也面临类似趋势，产品差异化将依赖定位和叙事。消费者购买的不仅是功能，更是确定感和价值认同。

从写代码到管 AI：开发者的角色正在被重新定义

开发者核心价值正从代码实现转向任务定义、质量约束及多执行体管理。AI Agent将承担更多编码工作，开发者需提升需求定义、边界设定、结果验证及系统协调能力，以驾驭AI完成复杂工程，推动软件工程重心上移。

业界资讯

国家数据局定调:Token官方中文名正式确认为“词元”

国家数据局正式将大模型基础计量单位Token定名为“词元”，并披露我国AI行业日均词元调用量突破140万亿次，此举终结业内译名争议，统一技术标准语境，降低沟通与统计成本，为后续政策制定和基础设施量化提供规范基础。

OpenAI 正在关闭其 Sora 视频生成应用程序

OpenAI宣布关闭Sora视频生成应用及相关API，原因是算力与战略重心转向世界模拟研究、机器人及更易盈利的企业级AI服务。Sora虽上线初期登顶应用商店，但用户安装和消费持续下滑，此次停运还导致迪士尼退出合作并取消10亿美元投资。

Claude Code 自动模式：一种更安全的跳过权限方式

Claude Code推出Auto mode，用模型分类器替代大量人工审批，在沙箱高维护与完全跳过权限高风险之间提供折中，借助输入层提示注入探测和输出层两阶段动作审查，自动放行大多数低风险操作并拦截越权、误判、数据外传等危险行为，实测将正常操作误拦率降至0.4%，但对真实危险操作仍有17%漏拦，适合替代无权限模式，不适合高风险场景的人审。

谷歌 TurboQuant 重磅发布：LLM 键值缓存内存压缩 6 倍、速度提升 8 倍，零精度损失、无需训练！

谷歌推出无训练向量量化算法TurboQuant，结合PolarQuant与QJL将大语言模型推理中的KV Cache内存需求压缩至少6倍，在Nvidia H100上把注意力计算提速最高8倍，并在长上下文测试中实现零精度损失，可直接用于现有模型，显著降低AI部署成本并加速长上下文应用落地。

使用 Gemini 3.1 Flash Live 构建实时对话智能体

谷歌通过 Live API 预览发布 Gemini 3.1 Flash Live，面向开发者构建低延迟实时语音与视觉代理，重点提升噪声环境下任务完成率、复杂指令遵循、对话自然度和90多种语言支持，更适合生产级实时对话应用，并可即刻在 Gemini API 与 AI Studio 集成使用。

将你的 AI 记忆和聊天记录带到 Gemini

Google为所有消费者账号上线迁移工具，用户可在Gemini设置页导入其他AI应用的记忆、偏好和聊天记录，方式包括复制提示词生成偏好摘要后粘贴到Gemini，或上传聊天记录ZIP文件。无需从零开始，Gemini可直接继承既有上下文并继续对话，相关功能正陆续推送。

佳文共赏

不要变成机器

作者批判社交媒体鼓吹的奋斗文化，认为其用极端工时、自我表演和流量话术绑架努力，把“投入”伪装成成功路径。真正有效的做法不是把自己变成只会硬扛的机器，而是围绕目标保持灵活、快速学习、按效率与质量做选择，守住个人真正的目的，停止崇拜苦熬式勤奋。

IDE 死了吗？

开发中心正从IDE内逐行编码转向“代理编排”：开发者更多负责定义意图、分派任务、监控进度、审查差异并合并结果，IDE退为调试、精查和最终修改工具；结论是IDE未死，但已失去主入口地位，软件开发正变为多代理协作下的监督、治理与审查工作。

人工智能会迫使代码进化，还是会让其走向消亡？

围绕“AI优先编程语言”出现多次实验，目标是提升LLM效率、减少歧义，但因生态、工具和社区壁垒，尚无实质采用；更现实的趋势是AI推动开发者转向Rust、TypeScript等强类型现有语言，以类型约束提升生成代码可靠性；至于跳过源码、让AI直出中间代码，业内认为仍属推测，人类审查与可解释性仍不可缺。

代码行数是有用的

作者反驳“代码行数毫无意义”的观点，认为代码行数是当前最可靠的复杂度代理，能较准确预测软件开发与维护成本及规模增长带来的负担；但它无法区分必要复杂度和偶发复杂度，因此适合做成本指标，不适合直接衡量个人生产力或价值，除非复杂度与价值关系长期稳定。

AI 编码助手并未加快交付，因为编码从来都不是瓶颈

AI编程助手提升了个人写码产出，却未显著加快项目交付，因为瓶颈已从编码转向需求定义、架构对齐与结果验证；行业数据也显示任务和PR增加同时评审时间大幅上升；结论是团队应把高质量规格与可验证验收标准作为核心产物，采用“灰盒”方式由人主导意图与验收，代码实现更多交给AI。

关于他*的慢下来的一些想法

作者认为编码代理在生产环境被过度放权，正加速制造脆弱、复杂且难维护的软件：代理会重复犯错、缺乏全局理解与有效检索，导致代码膨胀、测试失真、质量下滑，最终把团队和代码库逼入死角；现阶段应把代理限定在可局部验证、低风险的辅助任务中，由人主导架构、评审与最终质量关，放慢速度换取可维护性与控制权。

在中国人工智能生态圈内深耕两周后的感悟

作者考察中国AI生态后判断，硬件显著强于软件：深圳与大湾区供应链带来极快迭代和全球竞争力；软件层面模型、营收与爆款公司均落后西方，且估值已明显泡沫化。同时，中国创业者执行强但原创性偏弱，本地VC又偏好名校大厂履历，可能错过真正具全球潜力的少数异类创始人。

技术博客

你不知道的 Agent：原理、架构与工程实践

Agent 工程的核心不在更强模型，而在稳定循环外的工程设计：用 Harness 定义验证与边界，用分层上下文、按需 Skills、ACI 工具和记忆整合控制质量，用状态外化、评测、追踪和安全机制支撑长任务与多 Agent，并且决定成败的往往是验证、工具定义和基础设施，而非模型价格。

面向长期运行应用开发的 harness 设计

Anthropic通过为Claude引入规划、生成、评估多智能体架构，并用可评分标准、QA反馈和长时任务编排替代单智能体自评，显著提升了前端设计质量与端到端应用构建能力；让模型能在数小时内自主完成更完整、可用的全栈产品，但代价是更高的成本、时延与系统复杂度，且评估环节仍需随模型能力迭代简化。

代码智能体协同体系：多智能体编程高效运行的核心要素

开发正从单AI协作转向多代理编排，核心做法是用子代理或代理团队并行分工、共享任务与隔离上下文，再用计划审批、测试钩子、文件锁和人工审查建立质量门禁。当前多代理能显著提升吞吐与专业化，但真正瓶颈已从生成转向验证，成败取决于规格清晰度、协调机制和持续积累的AGENTS.md知识。

开源项目

LangGraph: 智能体编排框架

LangGraph是面向长运行、有状态智能体的底层编排框架，核心能力包括故障后精确恢复、执行中人工干预、短期与长期记忆、运行调试观测及生产级部署，可独立使用，也可与LangChain、LangSmith等集成，为构建、管理和规模化部署复杂智能体提供了通用基础设施。

last30days-skill：Claude Code 社区研究插件

last30days-skill 是面向 Claude Code 的近30天社区研究插件，聚合 Reddit、X、YouTube、TikTok、Instagram、Hacker News、Polymarket、Bluesky 等8个信号源，并行搜索、复合评分、跨平台趋同检测与补充搜索后，输出带真实引用的摘要或即用提示词。

MiniMax Skills：开发技能库

MiniMax Skills 是面向 AI 编程助手的开源开发技能库，整合前端、全栈、Android、iOS、着色器及文档处理等9项生产级能力，并提供 Claude、Cursor、Codex、OpenCode 的安装接入方案，目标是让 AI 代理稳定生成结构化、可落地的代码。

DeerFlow 2.0：字节跳动开源超级智能体框架

DeerFlow 2.0是字节跳动开源的超级智能体框架，基于LangGraph和LangChain重写，提供子代理编排、长期记忆、沙箱执行、技能扩展及IM集成，支持Docker与本地部署。
